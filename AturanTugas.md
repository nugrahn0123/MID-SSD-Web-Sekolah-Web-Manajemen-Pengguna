Ketentuan Presentasi Video YouTube dan Upload Proyek GitHub
Setiap kelompok wajib mengumpulkan proyek MID Semester dalam tiga bentuk utama, yaitu laporan proyek, video presentasi YouTube, dan repository GitHub. Ketiga komponen tersebut menjadi satu kesatuan penilaian dalam mata kuliah Scalable System Design.
1. Ketentuan Video Presentasi YouTube
Setiap kelompok wajib membuat video presentasi yang menjelaskan proyek Web Sekolah Terintegrasi Berbasis Scalable System Design. Video presentasi diunggah ke YouTube melalui salah satu akun anggota kelompok.
Video dapat diatur sebagai public atau unlisted. Jika menggunakan pilihan unlisted, pastikan link dapat dibuka oleh dosen tanpa meminta izin tambahan. Video tidak boleh diatur sebagai private, karena dosen berpotensi tidak dapat mengakses video tersebut.
Durasi video presentasi adalah 10 sampai 15 menit. Setiap anggota kelompok wajib tampil atau berbicara dalam video. Setiap anggota juga wajib menjelaskan bagian yang menjadi tanggung jawabnya.
Video presentasi minimal memuat bagian berikut:
Pembukaan dan pengenalan anggota kelompok
Latar belakang masalah
Tujuan proyek
Gambaran umum sistem web sekolah
Penjelasan modul yang dibuat
Arsitektur sistem
Pembagian vCPU atau server virtual
Rancangan database dan relasi tabel
Rancangan hak akses pengguna
Penjelasan unsur Scalable System Design
Risiko sistem dan solusi
Kesimpulan
Mahasiswa boleh menggunakan slide presentasi, rekaman layar, diagram arsitektur, ERD, tampilan prototipe web, atau kombinasi semuanya. Video harus memiliki suara yang jelas, tampilan yang terbaca, dan penjelasan yang runtut.
Mahasiswa tidak diperkenankan hanya membaca teks tanpa menjelaskan rancangan sistem. Penilaian akan memperhatikan pemahaman konsep, kejelasan penyampaian, kualitas arsitektur, dan kemampuan menghubungkan proyek dengan konsep Scalable System Design.
2. Ketentuan Upload Proyek ke GitHub
Setiap kelompok wajib mengunggah proyek web sekolah ke GitHub. Repository GitHub digunakan sebagai bukti pengerjaan proyek, dokumentasi teknis, dan media penilaian struktur sistem.
Repository dapat dibuat pada salah satu akun anggota kelompok. Nama repository disarankan menggunakan format berikut:
MID-SSD-Web-Sekolah-NamaKelompok
Contoh:
MID-SSD-Web-Sekolah-Kelompok-3
Repository GitHub minimal harus memuat:
Source code proyek web sekolah
File database atau rancangan database
File dokumentasi proyek
Diagram arsitektur sistem
Diagram relasi database atau ERD
File README.md
Daftar anggota kelompok dan pembagian tugas
Panduan instalasi dan cara menjalankan proyek
Link video presentasi YouTube
Screenshot tampilan sistem, jika ada
Repository boleh bersifat public atau private. Jika repository dibuat private, kelompok wajib mengundang dosen sebagai collaborator atau memberikan akses yang dapat dibuka oleh dosen. Repository public lebih disarankan agar proses pemeriksaan lebih mudah.
File README.md wajib memuat informasi berikut:
Judul proyek
Deskripsi singkat sistem
Nama anggota kelompok
Pembagian tugas setiap anggota
Daftar modul yang dibuat
Teknologi yang digunakan
Struktur folder proyek
Rancangan arsitektur sistem
Rancangan database
Cara instalasi
Cara menjalankan aplikasi
Akun login demo, jika ada
Link video presentasi YouTube
Penjelasan unsur Scalable System Design dalam proyek
Mahasiswa tidak diperkenankan mengunggah data sensitif ke GitHub, seperti password asli, file .env berisi kredensial, token API, private key, atau data pribadi siswa yang sebenarnya. Jika proyek membutuhkan file konfigurasi, gunakan file contoh seperti .env.example.
Jika proyek membutuhkan contoh data, mahasiswa wajib menggunakan data dummy. Data dummy dapat berupa nama siswa, guru, kelas, mata pelajaran, akun pengguna, dan data sekolah fiktif.
Commit history pada GitHub akan menjadi salah satu bahan pertimbangan dalam penilaian kontribusi kelompok. Setiap anggota disarankan melakukan commit sesuai bagian yang dikerjakan. Kelompok yang hanya memiliki satu commit besar pada akhir pengerjaan tetap dapat dinilai, tetapi akan lebih sulit menunjukkan kontribusi masing-masing anggota.
3. Format Pengumpulan
Setiap kelompok wajib mengumpulkan data proyek dengan format berikut:
Nama Kelompok:
Kelas:
Nama Anggota:
Judul Proyek:
Link YouTube:
Link GitHub:
Link Dokumen Laporan:
Ketiga link wajib dapat diakses oleh dosen pada saat batas akhir pengumpulan. Link yang tidak dapat dibuka dianggap belum memenuhi ketentuan pengumpulan proyek.
4. Output yang Wajib Dikumpulkan
Setiap kelompok wajib mengumpulkan tiga komponen utama:
Laporan proyek dalam format PDF
Link video presentasi YouTube
Link repository GitHub proyek web sekolah
Laporan proyek berisi rancangan sistem, arsitektur, pembagian vCPU, database, hak akses pengguna, alur kerja sistem, use case, unsur Scalable System Design, risiko sistem, solusi, dan kesimpulan.
Video YouTube berisi penjelasan proyek secara lisan oleh seluruh anggota kelompok.
Repository GitHub berisi source code, dokumentasi teknis, rancangan database, diagram, README, dan panduan menjalankan proyek.


5. Penilaian Video Presentasi
Penilaian video presentasi mencakup aspek berikut:
Kejelasan penjelasan proyek: 20%
Kelengkapan pembahasan modul sistem: 15%
Kualitas penjelasan arsitektur dan pembagian vCPU: 20%
Ketepatan penerapan konsep Scalable System Design: 25%
Kualitas diagram, slide, atau tampilan pendukung: 10%
Kontribusi dan keterlibatan setiap anggota kelompok: 10%
Video yang hanya menjelaskan tampilan web tanpa membahas arsitektur, database, integrasi layanan, skalabilitas, dan pembagian beban sistem tidak akan memperoleh nilai maksimal.
6. Penilaian Repository GitHub
Penilaian repository GitHub mencakup aspek berikut:
Kelengkapan source code proyek: 25%
Struktur folder dan kerapian kode: 15%
Kelengkapan README.md: 20%
Dokumentasi instalasi dan cara menjalankan sistem: 15%
Ketersediaan rancangan database, ERD, dan diagram arsitektur: 15%
Riwayat commit dan kontribusi anggota kelompok: 10%
Repository yang tidak dapat diakses oleh dosen sampai batas waktu pengumpulan dianggap belum memenuhi ketentuan pengumpulan proyek.
7. Catatan Penting
Mahasiswa tidak boleh hanya membuat tampilan web tanpa menjelaskan rancangan sistem. Proyek ini harus menunjukkan penerapan konsep Scalable System Design, terutama pada aspek pembagian modul, integrasi database, pembagian vCPU, strategi scaling, keamanan akses, monitoring, logging, dan optimasi database.
Mahasiswa juga tidak wajib melakukan deployment online. Untuk proyek MID, repository GitHub yang rapi, dokumentasi yang jelas, dan prototipe yang dapat dijalankan secara lokal sudah cukup. Deployment online dapat menjadi nilai tambahan apabila kelompok mampu melakukannya.

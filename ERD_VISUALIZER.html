🔗 Hubungan Antar Tabel

ROLES → USERS (1-to-Many)
Setiap role memiliki banyak user. Contoh: role Admin bisa memiliki 2+ user admin.
USERS → STUDENTS (1-to-Many)
Setiap user bisa memiliki akun student (optional).
USERS → GURUS (1-to-Many)
Setiap user guru akan terhubung ke 1 record di tabel GURUS.
USERS → GURU_BK (1-to-Many)
Setiap user guru BK akan terhubung ke 1 record di tabel GURU_BK.
CLASSES → STUDENTS (1-to-Many)
Setiap kelas memiliki banyak siswa. Contoh: Kelas X IPA 1 memiliki 35 siswa.
CLASSES → GURUS (Wali Kelas)
Setiap kelas memiliki 1 wali kelas (guru).
GURUS → TEACHING_JOURNALS (1-to-Many)
Setiap guru dapat membuat banyak jurnal mengajar (satu per hari per kelas).
CLASSES → TEACHING_JOURNALS (1-to-Many)
Setiap kelas memiliki banyak jurnal mengajar dari berbagai guru.
SUBJECTS → TEACHING_JOURNALS (1-to-Many)
Setiap mata pelajaran memiliki banyak jurnal mengajar.
STUDENTS → BK_CASES (1-to-Many)
Setiap siswa bisa memiliki banyak kasus konseling, pelanggaran, atau prestasi.
GURU_BK → BK_CASES (1-to-Many)
Setiap guru BK menangani banyak kasus siswa.
ACADEMIC_YEARS → CLASSES (1-to-Many)
Setiap tahun ajaran memiliki banyak kelas.
USERS → ACTIVITY_LOG (1-to-Many)
Setiap user memiliki banyak log aktivitas (audit trail).

💡 Tipe Relasi

  • 1-to-Many (||--o{): 1 parent bisa memiliki banyak child
  • Many-to-Many: Tidak ada dalam diagram ini, tapi bisa ditambah junction table jika diperlukan
  • Foreign Key Cascade: Jika parent dihapus, child ikut dihapus
  • Foreign Key Restrict: Tidak boleh menghapus parent jika masih ada child

💾 SQL Script untuk Membuat Database

📌 Instruksi

Copy seluruh script di bawah ini, lalu paste ke MySQL client (phpMyAdmin, MySQL Workbench, atau command line):

-- Membuat Database CREATE DATABASE IF NOT EXISTS school_system; USE school_system; -- Tabel ROLES CREATE TABLE roles ( id INT PRIMARY KEY AUTO_INCREMENT, name VARCHAR(50) NOT NULL UNIQUE, description TEXT, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ); -- Tabel USERS CREATE TABLE users ( id INT PRIMARY KEY AUTO_INCREMENT, role_id INT NOT NULL, name VARCHAR(100) NOT NULL, email VARCHAR(100) NOT NULL UNIQUE, password VARCHAR(255) NOT NULL, is_active BOOLEAN DEFAULT TRUE, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, FOREIGN KEY (role_id) REFERENCES roles(id) ON DELETE RESTRICT ); -- Tabel ACADEMIC_YEARS CREATE TABLE academic_years ( id INT PRIMARY KEY AUTO_INCREMENT, year INT NOT NULL, semester ENUM('1', '2') NOT NULL, start_date DATE NOT NULL, end_date DATE NOT NULL, is_active BOOLEAN DEFAULT FALSE, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, UNIQUE KEY uk_year_semester (year, semester) ); -- Tabel CLASSES CREATE TABLE classes ( id INT PRIMARY KEY AUTO_INCREMENT, name VARCHAR(50) NOT NULL, level VARCHAR(20) NOT NULL, academic_year_id INT NOT NULL, capacity INT DEFAULT 35, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, FOREIGN KEY (academic_year_id) REFERENCES academic_years(id) ON DELETE RESTRICT ); -- Tabel GURUS CREATE TABLE gurus ( id INT PRIMARY KEY AUTO_INCREMENT, user_id INT NOT NULL UNIQUE, nip VARCHAR(20) NOT NULL UNIQUE, nama VARCHAR(100) NOT NULL, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE ); -- Tabel GURU_BK CREATE TABLE guru_bk ( id INT PRIMARY KEY AUTO_INCREMENT, user_id INT NOT NULL UNIQUE, nip VARCHAR(20) NOT NULL UNIQUE, nama VARCHAR(100) NOT NULL, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE ); -- Tabel SUBJECTS CREATE TABLE subjects ( id INT PRIMARY KEY AUTO_INCREMENT, name VARCHAR(100) NOT NULL UNIQUE, kode VARCHAR(20) NOT NULL UNIQUE, jam_pelajaran INT DEFAULT 2, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ); -- Tabel STUDENTS CREATE TABLE students ( id INT PRIMARY KEY AUTO_INCREMENT, user_id INT NULL UNIQUE, class_id INT NOT NULL, nisn VARCHAR(20) NOT NULL UNIQUE, nama VARCHAR(100) NOT NULL, status ENUM('aktif', 'pindah', 'lulus', 'keluar') DEFAULT 'aktif', created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE SET NULL, FOREIGN KEY (class_id) REFERENCES classes(id) ON DELETE RESTRICT ); -- Tabel TEACHING_JOURNALS CREATE TABLE teaching_journals ( id INT PRIMARY KEY AUTO_INCREMENT, guru_id INT NOT NULL, class_id INT NOT NULL, subject_id INT NOT NULL, tanggal DATE NOT NULL, materi TEXT, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, FOREIGN KEY (guru_id) REFERENCES gurus(id) ON DELETE CASCADE, FOREIGN KEY (class_id) REFERENCES classes(id) ON DELETE CASCADE, FOREIGN KEY (subject_id) REFERENCES subjects(id) ON DELETE RESTRICT ); -- Tabel BK_CASES CREATE TABLE bk_cases ( id INT PRIMARY KEY AUTO_INCREMENT, siswa_id INT NOT NULL, guru_bk_id INT NOT NULL, tipe ENUM('konseling', 'pelanggaran', 'prestasi') NOT NULL, deskripsi TEXT NOT NULL, tanggal DATE NOT NULL, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, FOREIGN KEY (siswa_id) REFERENCES students(id) ON DELETE CASCADE, FOREIGN KEY (guru_bk_id) REFERENCES guru_bk(id) ON DELETE CASCADE ); -- Tabel ACTIVITY_LOG CREATE TABLE activity_log ( id INT PRIMARY KEY AUTO_INCREMENT, user_id INT NOT NULL, action VARCHAR(100) NOT NULL, model VARCHAR(100) NOT NULL, model_id INT, description TEXT, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE ); -- Membuat INDEX untuk performa -- Foreign Key Indexes CREATE INDEX idx_users_role ON users(role_id); CREATE INDEX idx_students_class ON students(class_id); CREATE INDEX idx_teaching_journals_guru ON teaching_journals(guru_id); CREATE INDEX idx_teaching_journals_class ON teaching_journals(class_id); CREATE INDEX idx_bk_cases_siswa ON bk_cases(siswa_id); CREATE INDEX idx_activity_user ON activity_log(user_id); -- Search Indexes CREATE INDEX idx_users_email ON users(email); CREATE INDEX idx_students_nisn ON students(nisn); CREATE INDEX idx_students_status ON students(status); SELECT 'Database created successfully!' as message;

  3. Copy-paste script di atas
  4. Klik "Execute" atau tekan Ctrl+Enter
  5. Jika berhasil, akan muncul pesan "Database created successfully!"
